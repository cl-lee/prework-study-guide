HTML

  • A HTML document starts with a !DOCTYPE declaration
  • Elements are building blocks in HTML that provide content and structure in a webpage.
  • An element contains everything starting from an opening tag to its closing tag
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The footer element is found at the bottom of a webpage

CSS

  • External style sheet changes the webpage style with a separate file that contains the CSS rules.
  • Internal style sheet defines style for a webpage, it can be found in the head section.
  • Inline style is applicable to an element only.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • A CSS Rule consists of a selector and a declaration block; a declaration block consists of property(s) and value(s).
  • Multiple selectors can be included in a CSS rule with comma(s)
  • By designating a class attribute as a selector, all elements with the assigned class will be affected by the CSS rule
  • In CSS, colours can be assigned by their semantic references (e.g. red, blue) or hexadecimal codes (e.g. #567890).

Git

  • git clone [copied SSH value]: clones the specified GitHub repository into the local environment
  • git status: checks what branch we are currently on, also shows changes made to the local repository
  • git checkout -b [branch-name]: creates a new branch and switches to it
  • git add -A: Adds all modified files in the current branch to the staging area
  • git commit -m [commit description]: Adds a message that describe what the commit contains
  • git pull origin main: Check if local branch matches the Github main branch
  • git push origin [branch-name]: push changes to the Github branch
